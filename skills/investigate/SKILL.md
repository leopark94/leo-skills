# /investigate — 경쟁 가설 진단

Anthropic 권장 디버깅 패턴. 5가지 이상 가설을 세우고 각각 독립 검증.

## 사용법

```
/investigate <문제 설명>
```

## 프로세스

### 1. 증상 수집
- 에러 메시지/로그
- 재현 단계
- 최근 변경사항 (`git log --oneline -10`, `git diff`)
- 환경 정보

### 2. 가설 수립 (최소 5개)

```markdown
| # | 가설 | 확률 | 검증 방법 |
|---|------|------|-----------|
| 1 | ... | 높음 | ... |
| 2 | ... | 중간 | ... |
| 3 | ... | 중간 | ... |
| 4 | ... | 낮음 | ... |
| 5 | ... | 낮음 | ... |
```

### 3. 순차 검증
- 확률 높은 것부터
- 각 가설 독립 검증 (한 가설 검증이 다른 가설 오염 방지)
- 검증 결과 기록

### 4. 수정
- 확인된 원인에 대해 최소 변경으로 수정
- 빌드 + 재현 테스트
- 회귀 확인

### 5. 보고

```markdown
## 진단 결과

### 원인
{확인된 근본 원인}

### 수정 내용
{변경 파일 및 내용}

### 검증
- [x] 원래 에러 해결
- [x] 빌드 통과
- [x] 회귀 없음

### 기각된 가설
- 가설 2: {기각 이유}
- ...
```
